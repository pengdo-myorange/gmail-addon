개인정보처리방침

최종 수정일: 2026년 4월 13일

"보내기 전에"(이하 "본 확장기능")는 사용자의 개인정보를 소중히 여기며, 다음과 같은 원칙으로 개인정보를 처리합니다.

1. 수집하는 정보

본 확장기능은 다음 정보를 처리합니다:

  • 이메일 본문 (일시적): 사용자가 [검토하기] 버튼을 클릭할 때, 작성 중인 이메일 본문이 검토를 위해 처리됩니다. 이메일 본문은 메모리에서만 일시적으로 처리되며, 본 확장기능에 의해 저장되지 않습니다.
  • 수신자 이름 (일시적): 검토 시 수신자 표시 이름(예: "김철수 과장")이 호칭 검증을 위해 처리됩니다. 이메일 주소는 전송되지 않으며, 어디에도 저장되지 않습니다.
  • 원본 메일 컨텍스트 (일시적, 답장 시): 답장/전달 시 원본 메일의 서두(첫 3줄)와 서명(마지막 5줄)이 호칭 검증을 위해 처리됩니다. 원본 메일 전체는 처리되지 않으며, 어디에도 저장되지 않습니다.
  • API 키: 사용자가 직접 입력한 Google Gemini API 키가 브라우저의 로컬 스토리지(chrome.storage.sync)에 저장됩니다.
  • 설정: 선택한 LLM 모델, 활성 검토 카테고리, 사용자 지정 규칙이 브라우저의 로컬 스토리지(chrome.storage.sync)에 저장됩니다.
  • 검토 이력 메타데이터: 검토 날짜, 발견된 이슈 수, 카테고리별 개수, 반영/무시 건수가 로컬에 저장됩니다. 이메일 내용, 제목, 수신자 정보는 포함되지 않습니다.
  • 사용 통계: 이벤트 유형(검토 시작, 수정 반영, 수정 무시, 오류)과 타임스탬프가 로컬에 저장됩니다. 외부로 전송되지 않습니다.

2. 외부 전송

사용자가 검토를 요청하면, 다음 데이터가 Google Gemini API로 전송됩니다:

  • 작성 중인 이메일 본문
  • 수신자 표시 이름 (호칭 검증 목적, 이메일 주소 제외)
  • 답장/전달 시: 원본 메일의 서두(첫 3줄)와 서명(마지막 5줄)

이 전송은 사용자 본인의 API 키를 사용하여 이루어지며, Google의 Gemini API 이용약관에 따라 처리됩니다.

본 확장기능의 개발자는 이메일 본문에 접근하거나 수집하지 않습니다. 자체 서버를 운영하지 않습니다.

3. 데이터 저장

  • API 키: chrome.storage.sync (사용자 브라우저 내부)
  • 검토 이력 / 사용 통계: chrome.storage.local (사용자 브라우저 내부)
  • 서버 저장: 없음. 본 확장기능은 자체 서버를 운영하지 않습니다.

4. 데이터 삭제

확장기능을 제거하면 모든 로컬 데이터(API 키, 검토 이력, 사용 통계)가 자동으로 삭제됩니다.

5. 제3자 제공

Google Gemini API를 통한 이메일 본문 전송을 제외하고, 어떠한 개인정보도 제3자에게 제공되지 않습니다.
