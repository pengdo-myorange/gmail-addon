# 보내기 전에 · BeforeSend

**Gmail 보내기 버튼을 누르기 직전, 단 한 번의 클릭으로**
**한국어 비즈니스 이메일의 오탈자·호칭·조사를 잡아주는 크롬 확장 프로그램**

[설치하기](#-설치-3단계) · [스크린샷](#-스크린샷) · [자주 묻는 질문](#-자주-묻는-질문) · [개인정보 처리](#-개인정보는-어떻게-다루나요)

## 이런 경험, 있으시죠?

> "보냄" 버튼을 누른 직후에
> **'부장님**님'**, 오탈자, 어색한 조사**를 발견합니다.
> 이미 상대방 받은편지함에 들어간 뒤입니다.

AI에게 "대신 써줘"라고 하기엔 내 말투가 사라지고,
문법검사기를 따로 돌리기엔 너무 귀찮습니다.

**보내기 전에**는 그 사이를 메웁니다.
내 문장은 그대로, **신뢰를 깎는 오류만** 잡습니다.

---

## 무엇이 다른가요?

|  | 보내기 전에 | Gmail 글쓰기 도우미 | 기존 맞춤법 검사기 |
|---|:---:|:---:|:---:|
| 내 문체 유지 | ✅ | ❌ (AI 문체로 덮어씀) | ✅ |
| 비즈니스 맥락(호칭·경어체) | ✅ | △ | ❌ |
| Gmail 안에서 바로 | ✅ | ✅ | ❌ (복사/붙여넣기) |
| 수정 전후 비교 미리보기 | ✅ | ❌ | △ |
| 서명·인용부 보호 | ✅ | — | ❌ (서식 망가뜨림) |
| 한 번 클릭 되돌리기 | ✅ | ❌ | ❌ |

> **핵심 철학**: "AI가 당신 대신 쓰는 게 아니라, 당신이 쓴 문장을 검토합니다."

**3. 선택적 반영 — 원하는 것만 체크**

각 항목에 체크박스와 **[이것만 반영]** 버튼이 있어서, 부담 없이 한 건씩 적용해볼 수 있습니다.

**4. 만약 마음에 안 들면 → 되돌리기 한 번**

적용 직후 7초간 떠 있는 `[되돌리기]` 토스트로 원상 복귀. 서명·인용부가 혹시라도 손상되면 **자동으로 롤백**됩니다.

## 무엇을 잡아내나요?

7개 카테고리를 검토합니다. 필요 없는 항목은 설정에서 끌 수 있습니다.

| 카테고리 | 예시 |
|---|---|
| 수신자 호칭 | `부장님님` → `부장님` |
| 중복 표현 | `역전앞` → `역 앞` |
| 띄어쓰기 | `할수있습니다` → `할 수 있습니다` |
| 오타/맞춤법 | `되요` → `돼요` |
| 경어체 불일치 | 한 메일 내 `~합니다`와 `~해요` 혼용 정리 |
| 조사 오류 | `책을이` → `책이` |
| 문단 구분 | 지나치게 긴 한 덩어리 → 자연스러운 단락 |

### 내 스타일도 추가할 수 있어요

설정에서 **사용자 규칙**을 자유롭게 입력할 수 있습니다.

```
"프로젝트" 대신 "과업"으로 쓴다
"일정" 대신 "마일스톤"이라는 표현을 쓴다
"~해주세요" 대신 "~ 부탁드립니다"로 바꾼다
```

## 🔒 신뢰할 수 있나요?

이메일은 받는 사람에게 **당신의 이름으로 도착하는 공식 문서**입니다. 그래서 이 확장은 과하게 고치지 않습니다.

- ✅ **서명·인용부는 절대 건드리지 않음** — DOM 추출부터 제외, 프롬프트에서도 수정 금지, 혹시 변경되면 자동 롤백
- ✅ **링크·굵기·글꼴·색상 등 서식 그대로** — 텍스트 교체 시 주변 스타일을 감싸고 있는 DOM 노드를 건드리지 않도록 설계
- ✅ **실행 취소 1초 이내** — 적용 후 7초간 `[되돌리기]` 버튼으로 원상 복귀
- ✅ **문체를 바꾸지 않음** — LLM이 전체 문장을 다시 쓰지 않고, 어순·어투 유지가 프롬프트에 내장

## 🚀 설치 (3단계)

### 1. 파일 다운로드

[최신 릴리스 페이지](https://github.com/pengdo-myorange/BeforeSend/releases/latest)에서 **`beforesend-vX.X.X.zip`** 을 내려받아 압축을 해제합니다.

> 또는 이 페이지 상단의 초록색 **Code** → **Download ZIP** → 압축 해제.

### 2. Chrome에 등록

1. 주소창에 `chrome://extensions` 입력 → 이동
2. 오른쪽 위 **개발자 모드** 토글을 켭니다
3. **압축 해제된 확장 프로그램 로드** 클릭
4. 압축 푼 폴더를 선택합니다

확장 목록에 **보내기 전에**가 나타나면 성공입니다.

### 3. API 키 연결

이 확장은 Google Gemini API를 **당신의 키로** 호출합니다. 개발자 서버를 거치지 않습니다.

1. [Google AI Studio](https://aistudio.google.com/apikey)에서 API 키를 발급받습니다 *(무료 할당량 있음)*
2. Chrome 툴바의 <img src="icons/icon16.png" width="16" height="16" alt="보내기 전에 아이콘"> **보내기 전에** 아이콘을 클릭해 설정 페이지를 엽니다
3. 키를 붙여넣고 **[연결 테스트]** → **[저장]**

이제 Gmail에서 메일을 작성하고 **[검토하기]**를 눌러보세요.

## ❓ 자주 묻는 질문

<details>
<summary><b>Q. Gemini 말고 다른 모델도 쓸 수 있나요?</b></summary>

현재는 Google Gemini만 지원합니다. 설정에서 **모델명을 직접 입력**할 수 있어 `gemini-3.1-flash-lite-preview`, `gemini-2.5-pro` 등 원하는 모델로 바꿀 수 있습니다.
</details>

<details>
<summary><b>Q. 이메일 내용이 어디로 가나요?</b></summary>

**[검토하기]** 버튼을 눌렀을 때만, **당신의 API 키로** Google Gemini API에 전송됩니다. 개발자 서버를 거치지 않으며, 어디에도 저장하지 않습니다. 자세한 내용은 [개인정보 처리 방침](privacy.html)을 참고해 주세요.
</details>

<details>
<summary><b>Q. AI가 다 써주는 게 더 편하지 않나요?</b></summary>

비즈니스 이메일은 **누가 썼는지가 곧 신뢰도**입니다. AI가 덮어쓴 문장은 매끄럽지만, 평소 내 말투와 어긋나면 오히려 어색합니다. 그래서 이 도구는 **문체는 그대로, 오류만** 잡습니다.
</details>

<details>
<summary><b>Q. 서명이 플레인 텍스트로 바뀐다거나, 글꼴이 망가질까 봐 걱정됩니다.</b></summary>

서명·인용부는 애초에 검토 대상에서 제외됩니다. 텍스트 교체도 주변 서식(`<b>`, `<span style>`, `<a>` 등)을 건드리지 않도록 설계돼 있고, 혹시라도 보호 블록이 달라지면 **자동으로 원상 복구**됩니다. 그래도 불안하면 적용 후 7초간 뜨는 **[되돌리기]** 버튼으로 한 번에 되돌릴 수 있습니다.
</details>

<details>
<summary><b>Q. 영어 이메일도 되나요?</b></summary>

한국어 전용입니다. 한국어 비즈니스 맥락(호칭, 경어체, 조사)을 정조준하고 있어서, 영어 메일에는 **[검토하기]** 버튼을 눌러도 "한국어 이메일만 검토할 수 있습니다" 안내가 표시됩니다.
</details>

<details>
<summary><b>Q. 답장/전달 메일도 되나요?</b></summary>

네. 원본 메일의 수신자·발신자 정보와 인용된 컨텍스트를 자동으로 인식해 검토에 활용합니다. 단 인용된 원문은 **수정하지 않습니다**(내가 새로 쓴 부분만).
</details>

<details>
<summary><b>Q. 회사 메일(Google Workspace)에서도 쓸 수 있나요?</b></summary>

네, `mail.google.com` 도메인이면 동작합니다. 다만 조직 정책으로 확장 프로그램 설치가 제한된 환경에서는 관리자 승인이 필요할 수 있습니다.
</details>

## 🛠 개인정보는 어떻게 다루나요?

- 이메일 본문은 **[검토하기]** 클릭 시에만 Google Gemini API로 전송됩니다
- 전송은 **사용자 본인의 API 키**로만 이루어지며, 개발자는 이메일 내용에 접근하지 않습니다
- 자체 서버 없음 — Gemini API 외 어떤 외부 서비스에도 데이터를 보내지 않습니다
- API 키와 설정은 브라우저 내부(`chrome.storage`)에만 저장됩니다
- 이메일 본문은 검토 후 **어디에도 저장되지 않습니다**
- 확장기능을 삭제하면 모든 로컬 데이터가 자동 삭제됩니다

전문: [`privacy.html`](privacy.html)

<summary><b>🧑‍💻 프로젝트 구조 (개발자용)</b></summary>

```
BeforeSend/
├── manifest.json          # Chrome 확장 설정 (Manifest V3)
├── background.js          # Service Worker — Gemini API 호출, 스토리지
├── content.js             # Gmail DOM 감지, 버튼 주입, 검토 오케스트레이션
├── selectors.js           # Gmail DOM 셀렉터 중앙 관리
├── prompts.js             # 오류 카테고리 정의, LLM 프롬프트
├── email-extractor.js     # 본문/인용부/수신자 컨텍스트 추출
├── text-replacer.js       # 가상 버퍼 + 앵커 맵 기반 서식 보존 교체
├── review-panel.js        # 모달 리뷰 패널 UI (Shadow DOM)
├── review-panel.css       # 리뷰 패널 스타일
├── options.{html,js,css}  # 설정 페이지
├── privacy.html           # 개인정보 처리 방침
└── icons/                 # 16/48/128px 아이콘
```

**기술 스택**

- Chrome Extension Manifest V3
- Google Gemini API (SSE 스트리밍)
- Shadow DOM (Gmail CSS 격리)
- MutationObserver + 폴링 (Gmail SPA 대응)
- WeakMap (다중 작성 창 상태 관리)
